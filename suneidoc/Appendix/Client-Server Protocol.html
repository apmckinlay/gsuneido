Client-Server Protocol

Suneido client-server communication is via a straightforward, mostly text-based protocol.

Note: Commands (e.g. admin) may be given in upper or lower case.

libraries => (name ...)

Returns a list of libraries in use on the server.

libget name => L# # ... \r\n text ...

Returns the definition(s) of a name from the libraries in use.

admin admin-request => OK

e.g. create, ensure, alter, drop

cursor query => C#

Returns a cursor# for the query.

transaction update | read => T#

Returns a transaction#.

tranlist => (# ...)

Return a list of outstanding transactions.

query T# query => Q#

Returns a query# for the query.

get +|- Q# | T# C#

rewind Q# | C#

output ( Q# | T# C# ) R# \r\n record => OK

update ( Q# | T# C# ) (field ...) K# R# \r\n key record => OK

header C | Q# => schema

order C | Q# => (field ...)

keys T# C | Q# => (field ...)

close C# => OK

request T# request => # of records

e.g. insert, update, delete

explain Q# => explanation

commit T#

abort T#

timestamp => a unique timestamp

dump tablename

copy

run code => value

text => OK

Enter text mode.

binary => OK

Enter binary mode.
