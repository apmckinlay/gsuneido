Use CallClass for Dialogs

Rather than put the Dialog call in the application code, possibly duplicated multiple times, put the Dialog code in the CallClass of the Controller.

For example:

Controller
    {
    CallClass(...)
        {
        return Dialog(0, Object(this, ...))
        }
    New(...)
        {
        ...
        }
    ...

This is especially useful when the Dialog call is more complex, e.g. using styles or sizes.

Note: This approach can also be used for Window as well as Dialog
