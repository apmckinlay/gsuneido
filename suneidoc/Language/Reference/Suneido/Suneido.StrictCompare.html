Builtin

Suneido.StrictCompare

(boolean)

StrictCompare starts as false. If StrictCompare is set to true, then comparisons (<, <=, >, >=) between different types will throw an exception.

The reason for this is that comparisons between types are ambiguous. You might write > 0 for positive numbers, but it also unintentionally includes e.g. dates. If it's intended for positive numbers, x > 0 can be rewritten as: Number?(x) and x > 0 (or use Positive?)

By, Gt, Min, object.Min, object.MinWith, Max, object.Max, object.MaxWith, object.Sort! are exempt from Suneido.StrictCompare

The goal is to eventually make StrictCompare always enabled.

See also: Cmp, Gt, Min, Max, object.Min, object.MinWith, object.Max, object.MaxWith
