string.SafeEval

() => value

Similar to string.Eval, but only handles constants, i.e. the result of Display.

Note: It is preferable to use SaveEval rather than Eval to avoid security issues.

If the string is not a constant, SafeEval will throw "invalid SafeEval"

For example:

"true".SafeEval() => true
"#(1, 2, 3)".SafeEval() => #(1, 2, 3)

Implemented using string.Compile

See also: string.ServerEval
